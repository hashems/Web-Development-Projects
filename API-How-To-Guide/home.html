World Time Engine API How-To Guide

by Sara Hashem

About this Guide

Welcome to my How-To Guide for the World Time Engine API.

If you're a world traveler and shiny new programmer like me, you already know juggling multiple time zones at once can be frustrating and confusing. And so can figuring out how to use APIs! Wouldn't it be nice to have immediate access to the current local time information about the current and surrounding time zones with an easy-to-use programmable interface? Imagine all the meetings (and flights) you could schedule with that kind of power!

Well, today's your lucky day because I'm here to tell you all about how to use this convenient API in an easy-to-use guide made especially for new programmers like us!

The World Time Engine API allows the user access to local time based on longitude and latitude coordinates as well as IP addresses. It's also a great way to access data such as local time in both Universal Time Coordinated and standard local formats, geographical coordinates, and time zone information.

The following pages will discuss:

  1. Obtaining an API key
  2. Location-Based Requests
  3. IP Address-Based Requests
  4. Getting Local Time
  5. Getting Time Zones
  6. JavaScript Dates and Times

Of course, if you're brave enough, the official documentation for this API can be found here. Feel free to use this guide in conjunction with the notes provided by the developers themselves.
