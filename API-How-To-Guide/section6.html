Getting Time Zones

In this final section, we will look at the various methods JavaScript has for dealing with date and time information.

It's a Date!

JavaScript allows you to create new Date objects using Date(). Creating a new Date object without any parameters will return the current date and time. There are many useful methods associated with Date object, but we will only look at a few that are directly relevant to the information returned by our API. You can find a more complete reference here.

For our purposes, we will be discussing the following Date object methods:

  • getTimeZoneOffset()
  • toJSON()
  • toString()

Getting Time Zone Offset

In the last section, recall how we were given UTC offsets for current and next time zones. These refer to the number of hours in either direction the local time deviates from Universal Coordinated Time. You can use the method getTimeZoneOffset() to compare time zones. This method returns the UTC offset in minutes, so yo properly compare this value to the value in your response, you will need to divide by 60. Notice that the sign is preserved after the division. Below is an example of how we might get the UTC offset of a given date and compare it to the response we got back from a request: 

				
var sampleDate = new Date();
var sampleOffset = sampleData.getTimeZoneOffset();

JSON Formatted Dates

Converting Date object strings to JSON format can be very useful moving date strings with other JSON formatted data. toJSON() returns a Date object string in JSON format. If you look closely at the repsonses we got back from our requests, you will notice how similar the formats of the dates and times are to the standard JSON format (YYYY-MM-DDTHH:mm:ss.sssZ). This format is in UTC time. Below is an example of this method in action: 

				
var sampleDate = new Date();
var sampleJSONDate = sampleData.toJSON();

The above lines of code might return something similar to 2015-11-30T16:51:54.927Z. Compared to the UTC time we got back from our IP address-based request (2015-11-26 05:58:04), you can see how you might go about comparing dates, times, or both to creat decision structures similar to the previous Date object method example.

It is important to note the T and Z characters in the JSON format. T indicates the starting position of the time, while Z indicates that the date and time are in UTC time.

Date Strings

UTC time and JSON formatting is all well and good for computers, but what if you want a more human friendly string to look at? toString() returns the day, date, time, UTC offset, and time zone of a Date object as a string. In fact, this is JavaScript's default method for displaying Date objects as strings. You would use this method as follows: 

				
var sampleDate = new Date();
var sampleJSONDate = sampleData.toString();

The lines above might return something like Tue Dec 01 2015 02:05:56 GMT+0900 (Korea Standard Time). One important thing to note is that the date and time are both local.

GMT Versus UTC

GMT, or Greenwich Mean Time, was the old standard for calculating and coordinating the world's time zones. You might find that some of the methods still use GMT as a point of reference, such as the toString() method we just looked at. However, keep in mind that JavaScript methods which rely on the old standard might become deprecated as the world makes the shift towards the newer UTC standard. For all human intents and purposes however, GMT and UTC essentially represent the same thing.
